# Definition of Done: Sprint 1 - Configuration Foundation
Generated: 2026-01-12

## P0: Configuration (Basic) Visualizer

### Must Pass
- [ ] Route `/cc-viz/configuration` loads without errors
- [ ] Sidebar "Configuration" is clickable (not disabled)
- [ ] Home page "Configuration" card is clickable with status "available"

**Settings Tab:**
- [ ] Model displayed in card format (e.g., "opus")
- [ ] Default mode displayed (e.g., "acceptEdits")
- [ ] Permissions grouped into categories: Bash, Tools, MCP, Other
- [ ] Permission search filters the list
- [ ] Plugins split into Enabled and Disabled sections
- [ ] Disabled plugins collapsible with "Show all" toggle

**CLAUDE.md Tab:**
- [ ] Markdown content rendered (headings, code blocks, lists)
- [ ] Section navigation shows detected sections (memory, personal-note, etc.)
- [ ] Clicking section scrolls to that content

**MCP Servers Tab:**
- [ ] Each server shown as a card
- [ ] Card displays: name, command, args, type
- [ ] Empty state if no servers configured

**Error Handling:**
- [ ] Missing settings.json shows "Not configured" message
- [ ] Missing CLAUDE.md shows helpful empty state
- [ ] Missing .mcp.json shows "No MCP servers configured"

### Verification Steps
1. Navigate to http://localhost:8182/cc-viz/configuration
2. Verify Settings tab loads with card-based layout
3. Search permissions for "git" - verify filter works
4. Click "Show all" on disabled plugins
5. Switch to CLAUDE.md tab - verify markdown renders
6. Switch to MCP Servers tab - verify server cards display
7. Rename ~/.claude/settings.json temporarily - verify graceful error
